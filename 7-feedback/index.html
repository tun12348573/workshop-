Sharing and Feedback

During my participation in the First Cloud AI Journey program at AWS-FCAJ, I had the opportunity to learn, practice, and experience a real working environment through the Mimi Jewelry E-commerce Platform project. This was a very valuable experience for me, helping me better understand the process of building an e-commerce system on AWS, from application development, infrastructure deployment, and functional testing to monitoring and resource cleanup.

Through this internship, I would like to share some of my feedback and personal thoughts as follows:

General Evaluation

1. Working environment

The working environment at AWS-FCAJ was friendly, open, and suitable for interns. During my participation, I felt that I had many opportunities to learn new knowledge, especially knowledge related to AWS Cloud, serverless architecture, and the process of deploying a real project.

The working environment encouraged me to proactively research, experiment, and solve issues by myself during the project. This helped me not only learn theory but also better understand how to apply knowledge in practice.

However, I think that having more experience-sharing sessions among interns or with previous participants would help everyone learn from each other more effectively.

2. Support from mentor and admin team

The mentor and admin team supported me a lot during the internship. When I encountered difficulties while working on the project, I was guided on how to analyze problems, identify errors, and come up with solutions instead of simply receiving direct answers.

This helped me improve my self-learning ability, problem-solving mindset, and debugging skills. In particular, while working on the Mimi Jewelry E-commerce Platform project, I learned how to check errors through CloudWatch Logs, API Gateway, Lambda, DynamoDB, Cognito, and other AWS services.

I highly appreciate the mentor’s practical guidance, which helped me understand why each step was necessary instead of only following commands.

3. Relevance between assigned work and my major

The internship work was highly related to my major in Software Engineering. The project required knowledge of frontend development, backend development, databases, authentication, APIs, deployment, and cloud computing.

Through the project, I was able to apply the knowledge I had learned at university, such as web programming, API design, data management, and functional testing. In addition, I also had the opportunity to learn new areas that I had not practiced deeply at school, such as:

  • Deploying backend services with AWS Lambda and API Gateway.

  • Storing data with Amazon DynamoDB.

  • Managing user authentication with Amazon Cognito.

  • Uploading product images to Amazon S3.

  • Distributing the website with Amazon CloudFront.

  • Monitoring errors with Amazon CloudWatch.

  • Configuring alerts with SNS.

  • Integrating payment deeply at school, such as:

  • Deploying backend services with AWS Lambda and API Gateway.

  • Storing data with Amazon DynamoDB.

  • Managing user authentication with ZaloPay Sandbox.

  • Managing infrastructure with AWS SAM and CloudFormation.

As a result, I gained a clearer understanding of how a real web system is deployed and operated in a cloud environment.

4. Learning opportunities and skill development

During the internship, I learned many important skills that support my career goal of becoming a full-stack developer and cloud developer.

In terms of technical skills, I improved my frontend development skills with Next.js, built backend services with Node.js/TypeScript, designed APIs, handled authentication, managed data in DynamoDB, and deployed systems to AWS.

In terms of deployment skills, I learned how to use AWS SAM to build and deploy backend resources, configure CloudFormation stacks, check outputs, deploy frontend files to S3, and create CloudFront invalidations.

In terms of debugging skills, I learned how to read logs, inspect requests in Developer Tools, debug API errors such as 401, 403, and 500, check CORS issues, verify Cognito tokens, and inspect IAM permissions.

In addition, I also practiced technical documentation skills by building workshop content, describing architecture, recording deployment steps, adding screenshots, and writing the resource cleanup section.

5. Culture and teamwork spirit

I felt the spirit of support and sharing throughout the program. When I faced problems, I could discuss them and receive suitable suggestions or guidance.

The working process helped me understand that communication and collaboration are very important in a real project. A software developer not only needs to know how to code but also needs to report progress, present problems clearly, receive feedback, and improve the product based on suggestions.

I also learned that when errors occur, it is important to stay calm and check each part of the system step by step instead of fixing issues in a hurry. This is a very necessary skill in a real working environment.

6. Policies and support for interns

The program created good conditions for interns to learn and practice with new technologies. Having the opportunity to work with AWS and build a project with relatively complete features gave me a more practical view of the software industry.

I think this program is very suitable for students who want to learn more about cloud, backend, frontend, and the process of deploying real systems.

What I was most satisfied with during the internship

What I was most satisfied with was being able to directly participate in building and deploying an e-commerce project on AWS from beginning to end.

Through the Mimi Jewelry E-commerce Platform project, I not only learned how to write code but also understood more about system architecture, how AWS services connect with each other, and how to deploy a complete website.

I was especially impressed by the experience of deploying the frontend to S3 and CloudFront, deploying the backend to Lambda, storing data in DynamoDB, handling login with Cognito, and processing payments with ZaloPay Sandbox. These experiences are very useful for my future career direction.

What I think the program could improve

In my opinion, the program could improve in the following areas:

  • Provide a weekly task checklist so interns can track their progress more easily.
  • Organize more code review or architecture review sessions so students can better understand the strengths and weaknesses of their projects.

These improvements would help interns have clearer direction, learn in a more structured way, and feel more confident when participating in real projects.

Would I recommend this internship to my friends?

If I have friends who are interested in cloud computing, web development, backend, frontend, or AWS, I would recommend them to join this program.

The reason is that the program helps students experience a real working environment, practice on a specific project, and learn technologies that are widely used in businesses.

Especially for Software Engineering students, understanding cloud deployment, serverless architecture, CI/CD, monitoring, and basic security is very important. This knowledge can give students an advantage when applying for internships or jobs in the future.

Suggestions and expectations

I hope the program will continue to be maintained and developed in the future so that more students can have the opportunity to learn and gain practical experience.

I also hope that in future programs, interns will have more opportunities to:

  • Join more in-depth workshops about AWS.
  • Receive additional guidance on system design and cloud architecture.
  • Practice more with CI/CD, security, and monitoring.

Other feedback

After the internship, I realized that I had learned many new things, not only about technical knowledge but also about working attitude, communication, and self-learning.

I understand that to become a good developer, besides knowing how to write code, I also need to improve many other skills such as problem-solving thinking, discipline, time management, English technical reading, report writing, and teamwork.

I would like to thank AWS-FCAJ, my mentor, and the admin team for giving me the opportunity to participate in this internship program. This was an important experience that gave me more motivation to continue learning, improving myself, and pursuing my career path in software engineering and cloud computing.
