### [Blog 1 - Security Response Automation on AWS](3.1-Blog1/)

This blog summarizes what I learned from the AWS Security Blog article about security response automation on AWS. The article explains how AWS services such as AWS CloudTrail, Amazon EventBridge, AWS Lambda, AWS Systems Manager, and Amazon SNS can work together to automatically detect and respond to security incidents. Through this blog, I learned how an event-driven architecture can reduce manual operations, shorten response time, and improve security operations in AWS environments.

### [Blog 2 - Automated Extraction of Compressed Files on Amazon S3 using AWS Batch and Amazon ECS](3.2-Blog2/)

This blog introduces an event-driven architecture for automatically extracting compressed files uploaded to Amazon S3. The solution uses Amazon S3, Amazon EventBridge, AWS Batch, Amazon ECS, and Amazon EBS to process large TAR files without maintaining a continuously running server. Through this blog, I learned why AWS Batch is more suitable than AWS Lambda for large file processing workloads and how containerized batch jobs can help optimize cost, scalability, and infrastructure management.

### [Blog 3 - Turning Amazon CloudWatch into a Proactive Monitoring Layer](3.3-Blog3/)

This blog discusses how Amazon CloudWatch can be used as more than just a tool for checking CPU usage or sending basic alerts. It explains how Metrics, Logs, Dashboards, and Alarms can be combined to build a proactive monitoring system. Through this blog, I learned that effective monitoring is not only about knowing when a system has failed, but also about detecting issues early, triggering automated responses, and reducing the time needed to resolve incidents.
